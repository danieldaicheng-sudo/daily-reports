## 🎯 问题解决与踩坑

**问题1：周报在飞书中的渲染失真**
- 现象：v3版周报加入架构图和流程图，但飞书代码块行号导致框线图混乱
- 根因：未提前考虑目标平台的渲染特性
- 解决：改为树状简洁风格（│├└▼►），仅用ASCII字符构建层级关系
- 教训：**输出前先在目标平台测试渲染效果**，不要假设所有平台都能完美呈现复杂格式

**问题2：外部搜索信息过载与筛选**
- 现象：税务筹划知识库的探索性搜索返回大量信息，难以判断深度边界
- 根因：缺乏明确的搜索停止标准
- 解决：建立"信源分级+Gap Check"双机制——第一优先级（监管官网）必须覆盖，第二优先级（专业机构）交叉验证，自媒体仅作消费者视角参考
- 教训：**搜索前预设信源优先级和校验底线**，避免无终点地"再搜一轮"

**问题3：多项目并行的进度同步**
- 现象：同一Project内存在两条工作线（销售Agent手册 vs IP语料项目），新会话启动时容易混淆上下文
- 根因：Project级别的知识库缺乏工作线隔离机制
- 解决：强制新会话启动时读取工作进度文件，明确当前所属工作线
- 教训：**多线并行时必须显性化当前工作上下文**，不能依赖隐性记忆

## 💡 方法论沉淀

**1. 周报四段式结构**（已验证可复用）
```
本周核心产出 → AI使用经验 → 下周重点计划 → 需团队支持
```
- 产出：数据说话，避免罗列版本号
- 经验：沉淀可复用的交互技巧（如PDF转文本的避坑指南）
- 计划：目标+节奏+验收标准+风险
- 支持：明确的协作者诉求，不是客气话

**2. 外部搜索七维框架**（税务知识库验证）
针对知识库建设类任务，七个并行搜索方向确保全面性：
1. 全球趋势（CRS/BEPS/全球最低税）
2. 内地税制最新动态
3. 香港税务体系（产品底层逻辑）
4. 产品×税务交集
5. 目标人群痛点
6. 政策落地执行现状
7. 跨境特殊规则

**3. 每日一句话标记机制**
解决AI无法推断主观价值的问题。每天收工时发送：
```
今天标记：完成了X，核心发现是Y；踩坑：Z
```
成本几乎为零，但周报质量显著提升。
