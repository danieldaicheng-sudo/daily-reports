# HEARTBEAT.md - 自动化工作日报/周报

## 任务调度规则

### 任务1：每日工作日报（21:00）
**触发条件**：每天 21:00
**执行内容**：
1. 读取 Chrome 浏览器中所有 Claude 标签页（通过 browser tabs + snapshot）
2. 分析当天工作内容，识别：
   - 今日完成的关键产出（不是罗列，而是提炼认知增量）
   - 重要数据/发现/决策
   - 待办事项状态变化
3. 生成结构化日报，包含：
   - 【今日核心认知】提炼今天工作带来的关键洞察
   - 【关键产出】价值导向的描述，说明"做了什么"和"为什么重要"
   - 【决策/调整】流程优化、策略调整等
   - 【明日待办】下一步行动
   - 【团队协作点】需要同步给团队的信息
4. **覆盖规则**：如 memory/YYYY-MM-DD.md 已存在，直接覆盖（重新生成最新版本）

### 任务2：推送日报到GitHub（21:05）
**触发条件**：每天 21:05（在日报生成后5分钟）
**执行内容**：
1. 检查 workspace 是否有未提交的变更
2. 添加 memory/ 目录下的日报文件到 git staging
3. 提交变更（commit message: "daily: YYYY-MM-DD 工作日报"）
4. 推送到 GitHub 远程仓库 origin master
5. 如遇冲突或推送失败，尝试强制推送（git push --force），如仍失败则记录到日志待人工处理

**前置条件检查**：
- 确保 git remote origin 已配置为 https://github.com/danieldaicheng-sudo/daily-reports.git
- 确保分支为 master（或根据实际情况调整为 main）

### 任务3：周六工作周报（18:00）
**触发条件**：每周六 18:00
**执行内容**：
1. 汇总本周 6 份日报
2. 提炼周度核心认知：
   - 本周工作的战略价值（对港险AI Agent项目的贡献）
   - 方法论沉淀（优化流程、工作规范迭代）
   - 数据/洞察积累（可用于团队知识库）
3. 生成团队对齐周报，包含：
   - 【本周核心成果】对项目的实际推进
   - 【关键认知/洞察】可以复用的方法论
   - 【下周计划】重点方向和预期产出
   - 【需团队协同事项】需要其他成员配合/知晓的内容

## 执行规范

1. **内容质量标准**：
   - 禁止单纯罗列工作事项
   - 必须提炼"这件事对项目/团队的价值"
   - 必须体现"我理解了什么，发现了什么"
   - 周报要有团队视角（其他人读了能获得什么信息）

2. **Chrome 访问**：
   - 使用 browser tabs 获取所有 Claude 页面
   - 使用 snapshot 读取每个页面的关键内容
   - 重点关注：
     * 完成的任务/达成的里程碑
     * 重要发现/数据/决策
     * 待办事项的变化

3. **输出格式**：
   - 日报：简洁的 Markdown，适合个人回顾
   - 周报：正式的 Markdown，适合团队共享

## 示例模板

### 日报示例结构
```
【2026-03-17 工作日报】

🧠 今日核心认知
[提炼今天的关键洞察，不是罗列]

✅ 关键产出
1. [产出名称] - [价值说明]
   - 关键数据/发现
   - 对项目的意义

🔄 流程/方法论
[如果有流程优化、工作方法调整]

⏳ 明日待办
[下一步行动]

👥 团队协同点
[需要团队知晓的信息]
```

### 周报示例结构
```
【2026年第X周 工作周报】

📊 本周核心成果
[对港险AI Agent知识库项目的实际推进]

💡 关键认知/方法论沉淀
[可以复用的经验、流程优化、新发现]

📈 数据/洞察积累
[有价值的数据、行业洞察、用户洞察]

🎯 下周计划
[重点方向和预期产出]

🤝 需团队协同
[需要其他成员配合/知晓的事项]
```
